# 云平台公网部署指南（满足现场公网演示要求）

> 课程要求：**必须在云平台部署**，答辩时通过 **公网 URL** 访问。仅 `localhost` 或 IDE 内运行会大幅减分。

## 一、推荐方案对比

| 方案 | 难度 | 费用 | 适合 |
|------|------|------|------|
| **A. 轻量云服务器 + Docker** | ★★☆ | 学生机约 50~100 元/年 | 最稳，老师验收最常见 |
| **B. 轻量云服务器 + PM2 + Nginx** | ★★★ | 同上 | 想自己掌控 Nginx/HTTPS |
| **C. PaaS（Zeabur 等）** | ★☆☆ | 有免费额度 | 不想买服务器 |

推荐使用方案 A：买一台国内云 **轻量应用服务器**（阿里云 / 腾讯云 / 华为云），用本项目自带的 `docker-compose.yml` 一键部署，公网 IP 直接访问。

## 二、方案 A：云服务器 + Docker（推荐）

### 1. 购买与开放端口

1. 登录 [阿里云](https://www.aliyun.com) 或 [腾讯云](https://cloud.tencent.com)，购买 **轻量应用服务器**（系统选 **Ubuntu 22.04**，1 核 2G 即可）。
2. 记下 **公网 IP**，例如 `123.45.67.89`。
3. 在控制台 **防火墙 / 安全组** 中放行：**80**（HTTP）、**443**（HTTPS，可选）、**22**（SSH，仅自己用）。

### 2. 上传代码

本地电脑安装 Git 后，将项目推到 GitHub/Gitee，在服务器上：

```bash
ssh root@你的公网IP
apt update && apt install -y git docker.io docker-compose-plugin
git clone https://gitee.com/你的用户名/FamilyBudgetBook.git
cd FamilyBudgetBook
```

也可使用 WinSCP / 宝塔面板 将整个项目文件夹上传到 `/root/FamilyBudgetBook`。

### 3. 修改配置并启动

编辑 `docker-compose.yml`，把 `JWT_SECRET` 改成随机长字符串（不要用默认值）：

```yaml
JWT_SECRET: 你的随机密钥至少32位
```

构建并后台运行：

```bash
docker compose up -d --build
```

首次启动会自动执行 `seed`，生成演示账号 **demo / 123456**。

### 4. 公网验证

浏览器访问（将 IP 换成你的）：

- 首页：`http://123.45.67.89`
- 健康检查：`http://123.45.67.89/api/health`

能打开登录页且能登录，即满足 **公网演示** 要求。

### 5. 填写提交材料

编辑项目根目录 `00-项目访问地址.txt`：

```
前端页面：http://123.45.67.89
后端 API：http://123.45.67.89/api
演示账号：demo / 123456
```

答辩前用手机 **4G/5G 网络**（不用教室 WiFi 连本机）再测一遍，避免“仅内网可访问”。

## 三、方案 B：云服务器 + PM2 + Nginx

适合已熟悉 Linux 的同学。

### 1. 安装环境

```bash
# 需 Node.js 22.5+
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs nginx
npm install -g pm2
```

### 2. 构建项目

```bash
cd /root/FamilyBudgetBook
cd frontend && npm install && npm run build && cd ..
cd backend && npm install && npm run seed && cd ..
```

### 3. 配置环境变量

```bash
cd backend
cp .env.example .env
nano .env
```

写入：

```
NODE_ENV=production
PORT=3000
HOST=0.0.0.0
JWT_SECRET=随机长密钥
DB_PATH=./data/family_budget.db
```

### 4. PM2 启动

在项目根目录：

```bash
# 先修改 deploy/ecosystem.config.cjs 里的 JWT_SECRET
pm2 start deploy/ecosystem.config.cjs
pm2 save
pm2 startup
```

### 5. Nginx 反向代理

```bash
cp deploy/nginx.conf /etc/nginx/sites-available/family-budget
# 编辑 server_name 为你的公网 IP 或域名
ln -s /etc/nginx/sites-available/family-budget /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

访问 `http://公网IP` 即可。

## 五、答辩现场检查清单

- [ ] 使用 **http://公网IP或域名** 打开，不是 localhost
- [ ] 手机流量能打开（证明非本机环回）
- [ ] 演示账号能登录，首页有统计数据与图表
- [ ] 收支记账、统计页（日/周/月/年）各演示 1 次
- [ ] 服务器已 `pm2` / `docker` 设置 **开机自启**（`restart: unless-stopped`）
- [ ] `00-项目访问地址.txt` 与 PPT 中 URL **一致**
- [ ] 可选：绑定域名 + 免费 HTTPS（Let’s Encrypt），URL 更正式

## 六、常见问题

**Q：访问公网 IP 打不开？**  
检查云厂商防火墙是否放行 80 端口；`docker ps` 或 `pm2 status` 是否在运行。

**Q：能开首页但登录失败？**  
在服务器执行：`cd backend && npm run seed`（Docker 用户：`docker exec -it family-budget node src/seed.js`）。

**Q：重启服务器后数据没了？**  
Docker 必须使用 `docker-compose.yml` 中的 `volumes` 持久化；PM2 方案保证 `DB_PATH` 指向固定目录。

**Q：必须用域名吗？**  
不必须，**公网 IP + 80 端口** 即可满足“公网访问”；有域名可额外加分。

## 七、与本项目代码的关系

- 生产模式设置 `NODE_ENV=production` 后，后端会托管 `frontend/dist` 静态资源，**同一端口** 提供页面与 `/api`，无需分开部署两个域名。
- 开发仍用 `localhost:5173` + `localhost:3000`，仅本地调试；**提交与答辩以云地址为准**。
