# 家庭记账本 - 接口设计文档

基础路径：`/api`  
认证方式：除注册/登录外，请求头携带 `Authorization: Bearer <token>`

## 1. 认证模块

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /auth/register | 注册（自动初始化成员与分类） |
| POST | /auth/login | 登录 |
| GET | /auth/me | 当前用户信息 |

**注册/登录请求体**
```json
{ "username": "demo", "password": "123456", "displayName": "张家" }
```

**响应**
```json
{ "token": "jwt...", "user": { "id": 1, "username": "demo", "displayName": "演示家庭" } }
```

## 2. 家庭成员

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /members | 成员列表 |
| POST | /members | 新增成员 |
| PUT | /members/:id | 编辑成员 |
| DELETE | /members/:id | 删除成员 |

## 3. 收支分类

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /categories?type=income\|expense | 分类列表 |
| POST | /categories | 新增分类 |
| PUT | /categories/:id | 编辑分类 |
| DELETE | /categories/:id | 删除分类 |

**新增分类**
```json
{ "name": "餐饮", "type": "expense", "sortOrder": 0 }
```

## 4. 收支记录

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /transactions | 分页查询 |
| POST | /transactions | 新增记录 |
| PUT | /transactions/:id | 编辑记录 |
| DELETE | /transactions/:id | 删除记录 |

**查询参数**：`type`, `memberId`, `categoryId`, `startDate`, `endDate`, `keyword`, `page`, `pageSize`

**新增记录**
```json
{
  "memberId": 1,
  "categoryId": 5,
  "amount": 35.5,
  "type": "expense",
  "transDate": "2026-06-01",
  "note": "午餐"
}
```

## 5. 汇总统计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /stats/summary | 收支汇总（文字描述） |
| GET | /stats/by-category | 按分类统计 |
| GET | /stats/by-member | 按成员统计 |
| GET | /stats/trend | 趋势数据（图表） |

**汇总 summary 参数**
- `period`: day | week | month | year
- `refDate`: 参考日期（默认今天）
- 或 `startDate` + `endDate` 自定义区间
- `memberId`: 可选，按成员筛选

**summary 响应示例**
```json
{
  "label": "2026年6月",
  "income": 12000,
  "expense": 3500.5,
  "balance": 8499.5,
  "summaryText": "统计区间：2026年6月；成员：全部成员。收入合计 ¥12000.00（1笔）..."
}
```

## 6. 数据库表（ER 概要）

- **users** ← 1:N → **family_members**
- **users** ← 1:N → **categories**
- **users** ← 1:N → **transactions**
- **transactions** → N:1 **family_members**, **categories**

详细表结构见 `backend/sql/init.sql`（MySQL 版）及 SQLite 自动建表逻辑（`backend/src/db.js`）。
